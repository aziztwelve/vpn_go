# Спецификации функциональности

Здесь хранятся спецификации всех фич VPN системы.

## Формат спецификации

Каждая спецификация должна содержать:

1. **Описание** - что делает фича
2. **Требования** - функциональные и нефункциональные
3. **Схема БД** - таблицы и связи
4. **API** - endpoints и контракты
5. **План реализации** - этапы разработки
6. **Оценка времени** - сколько займет разработка

## Список спецификаций

### Микросервисы

1. **[Auth Service](./01-auth-service.md)** - Авторизация и управление пользователями
   - Валидация Telegram initData
   - JWT токены
   - Управление ролями
   - **Оценка:** 13 часов (≈ 2 дня)

2. **[VPN Service](./02-vpn-service.md)** - Управление VPN серверами и устройствами
   - WireGuard ключи и конфигурации
   - IP адреса
   - Управление серверами
   - **Оценка:** 22 часа (≈ 3 дня)

3. **[Payment Service](./03-payment-service.md)** - Обработка платежей
   - Интеграция с ЮKassa/Stripe
   - Webhook обработка
   - Создание подписок
   - **Оценка:** 14 часов (≈ 2 дня)

4. **[Referral Service](./04-referral-service.md)** - Реферальная программа
   - Генерация ссылок
   - Начисление бонусов
   - Статистика
   - **Оценка:** 12 часов (≈ 1.5 дня)

5. **[Admin Service](./05-admin-service.md)** - Административные функции
   - Управление пользователями
   - Обработка заявок на вывод
   - Dashboard статистика
   - **Оценка:** 12 часов (≈ 1.5 дня)

6. **[Traffic Service](./06-traffic-service.md)** - Учёт трафика
   - Сбор статистики с WireGuard
   - Агрегация данных
   - Cleanup старых логов
   - **Оценка:** 11 часов (≈ 1.5 дня)

### Итоговая оценка

**Общее время разработки:** 84 часа (≈ 10.5 рабочих дней или 2 недели)

### Порядок разработки

Рекомендуемая последовательность:

1. **Auth Service** - базовая авторизация нужна всем
2. **VPN Service** - ядро функциональности
3. **Payment Service** - монетизация
4. **Referral Service** - маркетинг
5. **Traffic Service** - мониторинг
6. **Admin Service** - управление
