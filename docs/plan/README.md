# 📋 Plan — стратегия развития инфраструктуры

Эта папка — **долгосрочный план** (HA, масштабирование, отказоустойчивость), в отличие от `tasks/` (конкретные таски с реализацией).

## Документы

- [`00-ha-scaling-roadmap.md`](./00-ha-scaling-roadmap.md) — **главный roadmap** до 10 000+ юзеров: уровни 0 → 3, что делать и в каком порядке, чего НЕ делать (k8s, multi-master), стоимость, целевая архитектура.
- [`01-current-state-audit.md`](./01-current-state-audit.md) — снимок реального состояния прода (2026-05-03): что есть, чего нет, цифры по нагрузке, БД, DNS, бэкапам.
- [`02-postgres-backup.md`](./02-postgres-backup.md) — ✅ внедрено: hourly `pg_dump` локально + rsync на `nl01`. Архитектура, восстановление, мониторинг.
- [`03-health-checks.md`](./03-health-checks.md) — ✅ внедрено: `/live`, `/ready`, `/health` в Gateway с агрегацией gRPC Health v1 от всех апстримов и пингом БД.
- [`04-monitoring-plan.md`](./04-monitoring-plan.md) — 📝 план: Prometheus + Grafana + Loki + Alertmanager + Telegram. 4 категории метрик, алерты с runbook'ами, phased rollout по неделям. Не реализовано.

## Как этим пользоваться

1. Сначала читать **00 (roadmap)** — понять стратегию и принципы.
2. Потом **01 (audit)** — увидеть с какой точки стартуем.
3. Когда какая-то задача из roadmap'а превращается в работу — заводить отдельный файл в [`../tasks/`](../tasks/) с детальной реализацией.

## Связь с `tasks/`

| Plan | Tasks |
|---|---|
| Высокоуровневая стратегия, «что и почему» | Конкретные таски, «как именно» |
| Принципы, отказы от k8s/active-active | Реализация: миграции, скрипты, код |
| Roadmap на месяцы вперёд | Конкретные DoD на дни-недели |

Например: **уровень 1 → 1.6 «failover.sh»** в `00-ha-scaling-roadmap.md` ссылается на детали в [`../tasks/08-ha-backend-mirror.md`](../tasks/08-ha-backend-mirror.md).
